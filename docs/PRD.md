# 產品需求文件 (PRD) - 校園停車way查詢系統

## 1. 專案概述

### 背景與動機
為解決大量車輛在校園附近低速徘徊尋找車位所導致的交通壅塞問題（特別影響主要幹道與停車場出入口），同時減少車輛頻繁繞行帶來的交通事故風險，保障行人安全並維護周邊商圈動線順暢。

### 目標用戶
- 主要目標用戶：學生
- 其他潛在用戶：教職員、到訪校園之一般民眾、周邊商家顧客

### 核心價值主張
提供即時且智慧的停車與交通指引，改善校園周邊交通壅塞，達到「人車分流」的效果，除了確保行人安全外，同時結合大眾運輸並引導人流行經商店街，以活絡周邊商圈（如逢甲商圈）的經濟發展。

## 2. 功能需求

以下列出系統的主要功能與對應的使用者故事 (User Stories)：

### 1. 綜合停車管理服務
包含線上查看實時停車格剩餘資訊、線上申請停車場月租停車格、以及線上繳交停車費功能。
- **使用者故事**：作為一名 **學生**，我希望 **能線上查看實時停車格剩餘數量並直接完成繳費或申請月租**，以便 **我能快速找到車位並節省排隊繳費或辦理手續的時間**。

### 2. 智慧路況與路線規劃
提供即時路況查看功能，並能自動為車輛規劃避開壅塞路段的最佳行駛路線。
- **使用者故事**：作為一名 **學生駕駛**，我希望 **系統能根據即時路況自動為我規劃進入停車場的最佳路線**，以便 **避開塞車路段並安全抵達停車地點**。

### 3. 大型活動交通動態管理
配合校園大型活動進行進場、散場的車流引導與交通管制公告。
- **使用者故事**：作為一名 **學生或訪客**，我希望 **在學校舉辦大型活動時，能收到明確的交通管制與進散場路線指引**，以便 **我能在活動期間順暢地抵達與離開校園而不被塞在車陣中**。

### 4. 大眾運輸系統整合 (轉乘引導)
配合大眾運輸交通工具系統資訊，引導民眾將車輛停放在距離校區較遠的停車點，轉乘大眾運輸前往校園，減少校邊車流。
- **使用者故事**：作為一名 **學生或民眾**，我希望 **系統能推薦周邊替代停車場及大眾運輸轉乘方案**，以便 **當校園附近車位客滿時，我仍能將車停在較遠處並方便地搭車進入校園**。

### 5. 智慧行人路線規劃與商圈導客
自動規劃人行路線，避開高車流道路，盡可能選擇行人專用道與商店街，不僅落實人車分流保障安全，也能製造商機活化逢甲商圈。
- **使用者故事**：作為一名 **學生 (行人)**，我希望 **系統能幫我規劃一條安全且經過周邊商店街的步行路線**，以便 **我能在避開車潮、確保安全的同時，順道逛街購物**。

## 3. 非功能需求

### 技術限制
- 後端框架：Flask
- 模板引擎：Jinja2
- 資料庫：SQLite (可作為初期快速驗證使用的輕量資料庫)

### 效能與安全考量
- **即時性 (Real-time)**：停車場剩餘車位與路況資訊，建議要求資料更新延遲應在 30 秒以內，以避免使用者被錯誤資訊誤導。
- **安全性 (Security)**：停車費繳交與月租申請涉及金流與用戶個資，通訊過程須採用 HTTPS 加密傳輸，並確保第三方支付 API 串接的安全性。
- **高可用性 (High Availability)**：系統在大型活動進場/散場等高流量尖峰期間，需具備承受瞬間高併發流量的能力，避免系統崩潰關閉。

## 4. MVP 範圍 (Minimum Viable Product)

### Must Have (必要功能)
- 線上查看實時停車格剩餘資訊
- 即時路況與最佳行車路線自動規劃
- 基礎的介面與地圖呈現 (整合路況與停車場資訊)

### Should Have (應有功能)
- 線上繳交停車費與月租停車格申請
- 自動規劃人車分流之行人路線（引導至行人專用道或商店街）

### Nice to Have (加分功能)
- 大型活動交通管制推播與專屬進散場路線引導
- 大眾運輸與遠端停車轉乘方案推薦
- 商圈優惠券整合（與行人路線功能結合）
